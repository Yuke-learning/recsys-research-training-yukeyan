### 5. 执行共享 GPU 安全检查

启动任何 GPU 程序前，先通过 `nvidia-smi` 查询全部计算进程，并将每个 PID 映射到操作系统用户。

检查发现物理 GPU 0 和 GPU 1 已有其他任务，因此没有在这两张卡上叠加新任务。物理 GPU 2 没有计算进程，随后立即执行强制守卫：

```bash
~/.local/bin/shared-gpu-check --gpu 2
```

守卫确认 GPU 2 安全后，先运行 CUDA check-only，再进行正式训练。实验通过以下设置只暴露物理 GPU 2：

```bash
CUDA_VISIBLE_DEVICES=2
```

因此程序内部显示的 `cuda:0` 实际对应服务器物理 GPU 2。

## 安全复现方式

以后重新运行时使用带安全守卫的脚本：

```bash
ssh lab-gpu
cd /home/yukeyan/research-training/lesson03/experiment_20260715_113233

```

脚本会在每次新训练前重新检查全部计算进程、确认 PID 所有者，并执行 `shared-gpu-check`。如果目标 GPU 已被占用，脚本会停止而不是叠加任务。

## 问题与处理

- **已有 GPU 任务占用部分设备**：没有依据空闲显存直接选择设备，而是检查计算进程及其所有者，最终只使用安全的物理 GPU 2。
- **服务器缺少课程指定环境**：没有修改其他项目正在使用的环境，而是新建独立 `lesson03` Conda 环境。
- **PyTorch 初始化提示未安装 NumPy**：本实验实现除 PyTorch 外不依赖其他第三方包，数据处理、测试和 GPU 训练均正常完成，因此该提示未影响实验结果。
- **结果持久化要求**：除训练目录外，额外保存了环境导出、运行命令、GPU 守卫日志、输入输出校验和及压缩归档，并同步回本地。
